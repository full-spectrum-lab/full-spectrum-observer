## 5. 事务原子性
| 阶段 | 原子边界 | 规则 | 恢复 |
| --- | --- | --- | --- |
| Validate and execute | No DB transaction | Input/Adapter/Validation/Engine produce immutable candidate output in memory | Failure creates a failure Operation/Audit transaction; no success Observation |
| Write output artifact temp | File system | Write to Observer-owned temp; flush/fsync; calculate SHA-256 | On failure delete temp and mark STORE_FAILED |
| Promote content-addressed artifact | Atomic same-volume rename | Rename temp to artifacts/sha-prefix/sha256; verify final size/hash | Crash before DB commit may leave orphan; startup/maintenance removes unreferenced Observer-owned artifacts |
| Begin SQLite finalization | BEGIN IMMEDIATE | Acquire single Audit writer; verify current audit_head and instance lock | Lock timeout causes STORE_FAILED; no COMPLETED state |
| Persist evidence | Same SQLite transaction | Insert immutable RuntimeSnapshot, Artifact metadata, Observation final data, Idempotency result and final AuditEvent | Any SQL failure rolls back all DB rows |
| Commit | SQLite COMMIT | Only after final artifact exists and hashes match; Observation may become COMPLETED | If commit fails, artifact remains unreferenced and is safe to clean |
| Post-commit verify | Read-only | Confirm artifact exists/hash, Observation points to it, Audit head matches | Mismatch becomes integrity failure; never silently repair history |
