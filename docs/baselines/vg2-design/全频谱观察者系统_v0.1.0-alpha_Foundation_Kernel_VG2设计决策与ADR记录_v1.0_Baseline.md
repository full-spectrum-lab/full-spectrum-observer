| ADR-FK-002 | reason_code 命名空间 | 采用 13 个域前缀：REQ/INTAKE/ADAPTER/SCHEMA/GOV/SNAPSHOT/FACADE/ENGINE/OUTPUT/STORE/AUDIT/SEC/SYSTEM；代码稳定、消息可本地化、详情默认脱敏。 | 复用异常消息；单一 ERROR 前缀；HTTP 状态码式编号 | 可区分阶段和责任；不把自然语言当机器合同。 | reason_code 不随文案变化；新增仅向后兼容；删除需迁移。 | APPROVED_FOR_VG2 |
| ADR-FK-003 | Observer Host 语言与运行时 | C# / .NET 10 LTS，目标 net10.0，Windows x64 self-contained；模块化单体。 | Python 单体 Host；.NET 8；Node.js | 用户主技术栈为 C#；.NET 10 是当前 LTS；长期支持窗口优于 .NET 8；适合 Windows-first 与后续 Linux。 | 不把 Engine 算法移植到 C#；System 与 Engine 通过合同隔离。 | APPROVED_FOR_VG2 |
| ADR-FK-004 | Engine Facade 进程模型 | 每次分析启动一个受控 Python Worker 子进程；stdin/stdout 单行 JSON 协议；stderr 仅诊断；Worker 直接 import run_simulation；不启动 FastAPI，不使用 Engine SQLite。 | 进程内嵌 Python；长期驻留本地 HTTP 服务；本地 RPC/gRPC | 最小依赖、边界清楚、可超时和强制终止；避免 Engine API 自带存储与 System Evidence 所有权冲突。 | 进程启动开销可接受；v0.1 不承诺高吞吐。 | APPROVED_FOR_VG2 |
| ADR-FK-005 | 主要开发者入口 | 以 observer.exe CLI 为唯一外部入口；Application Use Case 为内部稳定接口；v0.1 不暴露 HTTP。 | CLI + HTTP 同时发布；只使用 Python CLI；本地 REST | 避免两套入口逻辑和额外攻击面；满足 Internal Alpha 和第二干净环境。 | 后续 HTTP 必须复用相同 Use Case，不得复制业务逻辑。 | APPROVED_FOR_VG2 |
| ADR-FK-006 | Canonical Serialization | Observer-owned Snapshot/Audit 使用 FS-OBS-CANON-1：UTF-8、键按 Unicode 排序、无空白、UTC 毫秒时间、禁止 NaN/Infinity、Audit payload 只存 digest/ref；Engine 输出使用 FSE-PYJSON-1，由固定 Python Worker sort_keys + compact JSON 生成并按原始字节 SHA-256。 | 所有对象统一 RFC 8785；普通 JSON pretty print；数据库行拼接 | 避免跨运行时浮点格式差异，同时保持 Observer Audit 的稳定规范化。 | serialization_id 必须进入 Snapshot/Audit/Manifest；算法变更需新版本。 | APPROVED_FOR_VG2 |
| ADR-FK-007 | CanonicalContext 保存策略 | Raw Input 与完整 CanonicalContext 默认不持久化；保存 input/context digest、ValidationResult、Snapshot、Output；内置 synthetic CASE 可因其随包公开而完整保留。 | 全部保存；全部不保存；加密保存所有输入 | 符合最小化和 v0.1 非 Exact Replay 边界；避免企业输入长期留存。 | v0.1 只声明 synthetic rerun / semantic reference，不声明 Exact Replay。 | APPROVED_FOR_VG2 |
| ADR-FK-008 | 本地存储实现 | Observer 自有 SQLite 单文件 observer.db + 文件 Artifact Store；表级逻辑所有权分离；WAL 禁用，journal_mode=DELETE，foreign_keys=ON，busy_timeout；不放网络共享。 | 多数据库文件；JSON 文件库；复用 Engine SQLite；嵌入式文档库 | Internal Alpha 最简部署；事务和唯一约束可满足幂等与 Audit；不耦合 Engine API 存储。 | Audit 表由触发器禁止 UPDATE/DELETE；Artifact 用 content-addressed path。 | APPROVED_FOR_VG2 |
| ADR-FK-009 | Audit Hash Chain | v0.1 使用单一 GLOBAL stream、单 Host 实例、进程内 SemaphoreSlim 单写者 + SQLite BEGIN IMMEDIATE；sequence_no 单调递增；genesis previous_hash=64 个 0。 | 按 Observation 分链；Merkle Tree；无链只签名每行 | 最易验证和解释；并发量低，串行写入成本可接受。 | 第二 Host 实例启动时检测锁并拒绝写入；v0.1 不支持集群。 | APPROVED_FOR_VG2 |
| ADR-FK-010 | 任务超时与取消 | 默认 30 秒，可配置 1-300 秒；Ctrl+C/取消令牌将状态置 CANCELLING，等待 2 秒后杀死 Worker 进程树；最终 CANCELLED 或 TIMED_OUT；不自动重跑 Engine。 | 完全不支持取消；只软取消；无限等待 | 子进程模型可可靠终止；避免无反馈挂起。 | Audit 记录取消来源、时间和是否强制终止。 | APPROVED_FOR_VG2 |
| ADR-FK-011 | Windows 与安装封装 | 首个包为 portable ZIP：.NET 10 win-x64 self-contained + Python 3.11 x64 private runtime + locked NumPy/Engine + Pack/Schemas/Docs/Licenses；测试支持 Windows 11 x64 与 Windows Server 2022 x64；Windows 10 非 LTSC 不作支持声明。 | MSI 安装器；要求用户预装 .NET/Python；单文件 PyInstaller | 离线、可复制、可校验，最适合第二干净环境；不引入安装器复杂度。 | 不得搜索 PATH 中任意 python.exe；必须使用包内绝对路径。 | APPROVED_FOR_VG2 |
